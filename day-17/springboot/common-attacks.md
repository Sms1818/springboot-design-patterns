# 🔥 1. CSRF (Cross-Site Request Forgery)

---

# 📌 What is CSRF?

---

CSRF attack tricks the browser into sending:

```text
Unwanted authenticated request
```

to a website where user is already logged in.

---

# 🧠 Core Idea

---

```text
User already authenticated
        ↓
Browser stores session cookie
        ↓
Attacker sends malicious link
        ↓
Browser automatically appends cookie
        ↓
Server thinks request is valid
```

---

# 🔥 Important Condition for CSRF

---

## 📌 CSRF mainly happens in:

```text
STATEFUL Authentication
(Session-based authentication)
```

Because browser automatically sends:

```text
JSESSIONID cookie
```

---

# 🔥 CSRF Flow Diagram

---

```text
User logs into bank application
        ↓
Browser stores JSESSIONID cookie
        ↓
Attacker sends malicious URL
        ↓
User clicks link
        ↓
Browser appends session cookie automatically
        ↓
Money transfer executed
```

---

# 🔥 Spring Security Config

---

# 💻 SecurityConfig.java

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(
            HttpSecurity http
    ) throws Exception {

        http.sessionManagement(session ->

                session.sessionCreationPolicy(
                        SessionCreationPolicy.IF_REQUIRED
                ))

                .authorizeHttpRequests(auth ->
                        auth.anyRequest().authenticated()
                )

                .formLogin(Customizer.withDefaults());

        return http.build();
    }
}
```

# 🔥 Why Browser Sends Cookie Automatically?

---

Because browser behavior is:

```text
Same domain request
→ append stored cookies automatically
```

# 🔥 How to Prevent CSRF?

---

# ✅ Solution → CSRF Token

---

# 📌 What is CSRF Token?

---

A unique token generated by server:

```text
Per user session
```

---

# 🧠 Flow

---

```text
Server creates CSRF token
        ↓
Token embedded into forms
        ↓
Browser sends token
        ↓
Server validates token
        ↓
Only valid request accepted
```

---

# 🔥 Why JWT APIs Usually Disable CSRF?

---

Because JWT APIs are mostly:

```text
STATELESS
```

and do not use:

```text
Session cookies
```

# 🔥 2. XSS (Cross-Site Scripting)

---

# 📌 What is XSS?

---

XSS allows attacker to inject:

```text
Malicious JavaScript
```

into webpages viewed by other users.

---

# 🧠 Common Targets

---

- Comment sections
- Chat systems
- Forums
- Search fields
- User profiles

---

# 🔥 Main Goal of XSS

---

Attackers usually try to:

```text
Steal session cookies
Hijack sessions
Modify webpage
Redirect users
Execute malicious JS
```

---

# 🔥 XSS Flow

---

```text
Attacker submits malicious script
        ↓
Script stored in DB
        ↓
Another user opens page
        ↓
Browser executes script
        ↓
Attacker steals session/cookies
```

---

---

Because cookie contains:

```text

```

```text
Authenticated user session
```

# 🔥 How to Prevent XSS?

---

# ✅ Escape User Input

---

Convert:

```html

```

into:

```html

```

---

# ✅ Validate User Input

---

Reject:

- Scripts
- Dangerous HTML
- Invalid characters

---

# ✅ Avoid Rendering Raw HTML

---

Avoid:

```html

```

```html

```

because it escapes HTML automatically.

# 🔥 3. CORS (Cross-Origin Resource Sharing)

---

# 📌 Important

---

CORS is:

```text
NOT an attack
```

It is actually:

```text
Browser security mechanism
```

---

# 🔥 What is Origin?

---

Origin means combination of:

```text
Protocol + Domain + Port
```

---

# 🔥 Different Origin Examples

---

| Client | Server | Different? |
|---|---|---|
| https://localhost:8080 | http://localhost:8080 | ✅ |
| https://localhost:8080 | https://localhost:9090 | ✅ |
| https://localhost:8080 | https://sub.localhost:9090 | ✅ |

---

# 🧠 Why CORS Exists?

---

To prevent malicious websites from making requests to another server.

---

# 🔥 CORS Flow

---

```text
Frontend tries API call
        ↓
Browser checks origin
        ↓
Different origin detected
        ↓
Browser checks server CORS headers
        ↓
If allowed → request succeeds
Else → blocked
```

---

# 🔥 Spring Security CORS Config

---

# 💻 SecurityConfig.java

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain
    securityFilterChain(HttpSecurity http)
            throws Exception {

        http.sessionManagement(session ->

                session.sessionCreationPolicy(
                        SessionCreationPolicy.IF_REQUIRED
                ))

                .authorizeHttpRequests(auth ->
                        auth.anyRequest().authenticated()
                )

                .cors(cors ->

                        cors.configurationSource(
                                request -> {

                                    CorsConfiguration config =
                                            new CorsConfiguration();

                                    config.setAllowedOrigins(
                                            List.of(
                                                    "https://sub.localhost:9090"
                                            )
                                    );

                                    config.setAllowedMethods(
                                            List.of(
                                                    "GET",
                                                    "POST",
                                                    "PUT",
                                                    "DELETE"
                                            )
                                    );

                                    config.setAllowedHeaders(
                                            List.of(
                                                    "Authorization",
                                                    "Content-Type"
                                            )
                                    );

                                    config.setAllowCredentials(true);

                                    return config;
                                }
                        )
                )

                .formLogin(
                        Customizer.withDefaults()
                );

        return http.build();
    }
}
```

---

# 🔥 Important CORS Headers

---

| Header | Purpose |
|---|---|
| Access-Control-Allow-Origin | Allowed frontend |
| Access-Control-Allow-Methods | Allowed HTTP methods |
| Access-Control-Allow-Headers | Allowed headers |
| Access-Control-Allow-Credentials | Allow cookies/auth |

# 🔥 4. SQL Injection

---

# 📌 What is SQL Injection?

---

Attacker injects malicious SQL into user input.

---

# 🧠 Goal

---

- Bypass authentication
- Read sensitive data
- Delete tables
- Modify records

---

# 🔥 Unsafe Query Example

---

# 💻 Controller

```java
@GetMapping("/find")
public List<UserDetails> findUser(
        @RequestParam String name
) {

    return userDetailsService.findByName(name);
}
```

---

# 💻 Vulnerable Service

```java
public List<UserDetails> findByName(
        String name
) {

    String sql =

            "SELECT * FROM user_details " +
            "WHERE user_name = '" +
            name +
            "'";

    return entityManager
            .createNativeQuery(
                    sql,
                    UserDetails.class
            )
            .getResultList();
}
```

---

# 🔥 Dangerous Input

---

```text
AA' OR '1'='1
```

---

# 🧠 Generated SQL

---

```sql
SELECT *
FROM user_details
WHERE user_name = 'AA'
OR '1'='1'
```

---

# ❌ Result

---

```text
Entire table returned
```

because:

```sql
'1'='1'
```

is always true.

---

# 🔥 SQL Injection Flow

---

```text
User input directly concatenated
        ↓
SQL query manipulated
        ↓
Database executes malicious query
```

---

# 🔥 How to Prevent SQL Injection?

---

# ✅ Parameterized Query

---

# 💻 Safe Query

```java
public List<UserDetails> findByName(
        String name
) {

    String sql =

            """
            SELECT *
            FROM user_details
            WHERE user_name = :name
            """;

    return entityManager
            .createNativeQuery(
                    sql,
                    UserDetails.class
            )

            .setParameter("name", name)

            .getResultList();
}
```

---

# 🧠 Why Safe?

---

Because input is treated as:

```text
DATA
```

NOT:

```text
SQL CODE
```

---

# 🔥 Internal Flow

---

```text
Query compiled first
        ↓
Input bound separately
        ↓
Input cannot change SQL structure
```
